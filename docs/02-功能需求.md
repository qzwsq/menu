# 02 - 功能需求

## 2.1 每周菜单管理

### 2.1.1 菜单列表
- 展示已创建的每周菜单，按周倒序排列
- 显示周范围（周一~周日）、就餐人数、状态（草稿/已确认）
- 支持创建、删除、查看/编辑操作

### 2.1.2 创建/编辑菜单
- **选择周次**：支持选择当周及未来任意周（以周一为起点）
- **按天编排**：一周七天（周一~周日），每天包含：
  - 可多个主食项（如米饭、馒头、面条等）
  - 可多个菜品项（按自定义分类筛选，如荤菜、素菜、汤等）
- **就餐人数**：设置当周就餐人数，影响后续采购清单计算
- **历史输入复用**：主食和菜品的输入框支持下拉选择，选项来自本地食物库中已保存的记录

### 2.1.3 数据规范
- 同一天允许添加多个主食和多个菜品
- 主食和菜品均来自统一食物库，避免重复录入
- 菜单编辑保存时校验：每天至少包含1个菜品

## 2.2 菜品库管理

### 2.2.1 菜品列表
- 展示所有已保存的菜品，支持按分类筛选
- 支持搜索（按名称）
- 支持新增、编辑、删除

### 2.2.2 新增菜品
- **手动创建**：输入菜品名称、选择分类
- **LLM自动生成**（推荐）：
  - 用户输入菜品名称 → 后端调用LLM生成菜品配置
  - LLM返回：食材清单（名称+用量+单位）、营养数据（每份热量/蛋白质/脂肪/碳水等）
  - 用户可确认、修改后保存
- 后续补充食材时也支持LLM辅助生成

### 2.2.3 菜品详情/编辑
- 展示菜品基本信息（名称、分类）
- 展示关联食材清单：食材名称、用量、单位
- 营养数据（每份）：热量(kcal)、蛋白质(g)、脂肪(g)、碳水(g) 等
- 支持手动编辑食材清单和营养数据
- 支持LLM重新生成/补充

### 2.2.4 食材配置单条数据结构
```
{
  ingredient_name: String,   // 食材名称（如：猪里脊肉）
  quantity: Float,           // 用量
  unit: String,              // 单位（克/g、个、ml）
}
```

## 2.3 采购清单

### 2.3.1 生成采购清单
- **触发条件**：选择一份已创建的周菜单 + 输入就餐人数
- **计算逻辑**（后端）：
  1. 获取菜单中所有菜品的食材配置
  2. 按食材合并同类项，用量 × 人数系数
  3. 自动将食材归类到分组（如肉类、蔬菜、干货、调料等）
- **返回数据**：按分类分组的食材清单，每组包含食材项列表

### 2.3.2 App内Todo清单展示
- **展示形式**：在采购清单Tab内按分类展示为Todo清单
- 每条食材项包含：名称、总用量、单位、来源菜品
- 每条食材项支持勾选（checkbox），标记已购买
- 支持按分类展开/折叠
- 勾选状态保存在前端本地，切换页面不丢失

## 2.4 营养评估

### 2.4.1 营养计算
- **输入**：指定一周菜单 + 就餐人数
- **计算过程**：
  1. 汇总菜单中所有菜品每份营养数据 × 份数
  2. 得出每日平均营养摄入
  3. 调用LLM对营养数据进行评估
- **输出指标**：
  - 日均热量(kcal)、蛋白质(g)、脂肪(g)、碳水(g)
  - 与推荐摄入量对比
  - 异常提示（如热量过高/过低、蛋白质不足等）

### 2.4.2 营养报告
- 可视化展示周营养摄入趋势（柱状图/折线图）
- LLM生成文字评估建议
- 支持导出为PDF

## 2.5 分类管理

- 默认分类：主食、荤菜、素菜、汤、凉菜
- 用户可自定义添加/修改/删除分类
- 每个分类关联类型：主食(staple)/菜品(dish)
- 支持排序
