# 04 - 数据库设计

## 4.1 概述

- **前端 SQLite (sqflite)**：存储用户的菜单、食物库、分类、设置等日常数据
- **后端 MySQL**：存储LLM生成的菜品模板（食材配置、营养数据），作为公共模板库

---

## 4.2 SQLite 表设计 (Flutter本地)

### 4.2.1 categories — 自定义分类

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID |
| name | TEXT | NOT NULL | 分类名称 |
| food_type | TEXT | NOT NULL | staple(主食) / dish(菜品) |
| sort_order | INTEGER | DEFAULT 0 | 排序权重 |
| created_at | TEXT | NOT NULL | ISO8601 创建时间 |

### 4.2.2 foods — 食物库

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID |
| name | TEXT | NOT NULL UNIQUE | 食物名称 |
| food_type | TEXT | NOT NULL | staple / dish |
| category_id | TEXT | FK → categories.id | 所属分类 |
| ingredients | TEXT | DEFAULT '[]' | JSON数组，食材配置列表 |
| nutrition | TEXT | DEFAULT '{}' | JSON对象，营养数据(每份) |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

**ingredients JSON 结构：**
```json
[
  {"name": "猪里脊肉", "quantity": 200, "unit": "g"},
  {"name": "青椒", "quantity": 2, "unit": "个"}
]
```

**nutrition JSON 结构：**
```json
{
  "calories": 350,
  "protein": 25,
  "fat": 18,
  "carbs": 15,
  "fiber": 3,
  "sodium": 800
}
```

### 4.2.3 weekly_menus — 每周菜单

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID |
| name | TEXT | | 菜单名称（可选） |
| week_start | TEXT | NOT NULL | 周一日期 YYYY-MM-DD |
| week_end | TEXT | NOT NULL | 周日日期 |
| diner_count | INTEGER | DEFAULT 1 | 就餐人数 |
| status | TEXT | DEFAULT 'draft' | draft / confirmed |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

### 4.2.4 menu_meals — 菜单菜品明细

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID |
| menu_id | TEXT | FK → weekly_menus.id ON DELETE CASCADE | 关联菜单 |
| day_of_week | INTEGER | NOT NULL | 1-7 (周一~周日) |
| food_id | TEXT | FK → foods.id | 关联食物 |
| food_name | TEXT | NOT NULL | 食物名称（冗余，方便展示） |
| food_type | TEXT | NOT NULL | staple / dish |
| meal_order | TEXT | DEFAULT 'lunch' | lunch / dinner / snack |
| servings | REAL | DEFAULT 1 | 份数 |
| sort_order | INTEGER | DEFAULT 0 | 当日排序 |

### 4.2.5 settings — 用户设置

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| key | TEXT | PK | 设置键 |
| value | TEXT | | 设置值 |

## 4.5 建表SQL (参考)

### SQLite
```sql
CREATE TABLE categories (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    food_type TEXT NOT NULL CHECK(food_type IN ('staple', 'dish')),
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL
);

CREATE TABLE foods (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    food_type TEXT NOT NULL CHECK(food_type IN ('staple', 'dish')),
    category_id TEXT REFERENCES categories(id),
    ingredients TEXT DEFAULT '[]',
    nutrition TEXT DEFAULT '{}',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

CREATE TABLE weekly_menus (
    id TEXT PRIMARY KEY,
    name TEXT,
    week_start TEXT NOT NULL,
    week_end TEXT NOT NULL,
    diner_count INTEGER DEFAULT 1,
    status TEXT DEFAULT 'draft' CHECK(status IN ('draft', 'confirmed')),
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

CREATE TABLE menu_meals (
    id TEXT PRIMARY KEY,
    menu_id TEXT NOT NULL REFERENCES weekly_menus(id) ON DELETE CASCADE,
    day_of_week INTEGER NOT NULL CHECK(day_of_week BETWEEN 1 AND 7),
    food_id TEXT NOT NULL REFERENCES foods(id),
    food_name TEXT NOT NULL,
    food_type TEXT NOT NULL CHECK(food_type IN ('staple', 'dish')),
    meal_order TEXT DEFAULT 'lunch' CHECK(meal_order IN ('breakfast', 'lunch', 'dinner', 'snack')),
    servings REAL DEFAULT 1,
    sort_order INTEGER DEFAULT 0
);

CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

### MySQL
```sql
CREATE TABLE dish_templates (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    food_type VARCHAR(20) NOT NULL,
    category VARCHAR(50),
    ingredients JSON NOT NULL,
    nutrition JSON NOT NULL,
    created_by_llm TINYINT(1) DEFAULT 0,
    llm_model VARCHAR(100),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_name (name),
    INDEX idx_food_type (food_type),
    INDEX idx_category (category)
);
```
