# 08 - 开发计划

## 8.1 分期规划

```
Phase 1 ────────► Phase 2 ────────► Phase 3 ────────► Phase 4
基础框架         核心功能          衍生功能          打磨上线
(1周)            (2-3周)           (2周)             (1周)
```

---

## 8.2 Phase 1: 基础框架搭建

**目标：** 前后端项目初始化，数据库就绪，可运行的基础骨架

### 后端任务
| 任务 | 说明 |
|------|------|
| FastAPI 项目初始化 | 项目结构、依赖管理、配置加载(config.yaml) |
| MySQL 建表 | 执行 dish_templates 建表SQL |
| SQLAlchemy 模型 | ORM 模型定义 + 异步session |
| LLM Service 骨架 | 统一LLM调用接口 + OpenAI/Ollama适配 |
| 健康检查API | `GET /api/health` |

### 前端任务
| 任务 | 说明 |
|------|------|
| Flutter 项目初始化 | `flutter create` + 目录结构搭建 |
| 主题和路由 | Material 3 主题 + GoRouter 路由配置 |
| SQLite 初始化 | DatabaseHelper + 建表 |
| API Client 封装 | Dio 封装 + 拦截器 |
| 页面骨架 | 首页、菜单、菜品、采购、营养、设置 空页面 |

### 交付物
- 后端可启动，健康检查API可访问
- Flutter App 可在模拟器运行，页面间可导航

---

## 8.3 Phase 2: 核心功能

**目标：** 菜品库 + 每周菜单完整可用

### 后端任务
| 任务 | 说明 |
|------|------|
| POST /api/dishes/generate | LLM生成菜品配置 |
| GET/POST/PUT/DELETE /api/dishes | 菜品模板 CRUD |
| GET /api/dishes/search | 菜品搜索 |
| LLM Prompt 调优 | 食材生成 + 营养数据生成 Prompt |

### 前端任务
| 任务 | 说明 |
|------|------|
| 分类管理 CRUD | 分类列表/新建/编辑/删除页面 |
| 菜品列表页 | 列表展示 + 分类筛选 + 搜索 |
| 新建菜品页(LLM) | 输入菜名 → 调用LLM → 确认保存 |
| 菜品详情/编辑页 | 食材清单编辑 + 营养数据编辑 |
| 食物下拉选择器组件 | 主食/菜品 下拉选择 + 自动创建 |
| 菜单列表页 | 菜单列表 + 创建/删除 |
| 菜单编辑页 | 按天编排主食+菜品 + 切换日期 |
| SQLite DAO 完整实现 | foods / categories / menus / meals |

### 交付物
- 用户可创建菜品（含LLM生成食材配置）
- 用户可编排完整周菜单

---

## 8.4 Phase 3: 衍生功能

**目标：** 采购清单Todo + 营养评估报告

### 后端任务
| 任务 | 说明 |
|------|------|
| POST /api/shopping/generate | 采购清单计算 + 自动分类分组 |
| POST /api/nutrition/analyze | 营养汇总计算 |
| POST /api/nutrition/assess | LLM营养评估 |
| PDF导出模板 | 营养报告PDF模板 |

### 前端任务
| 任务 | 说明 |
|------|------|
| 采购清单页面 | 按分类展示Todo清单 + 勾选/展开折叠 |
| 营养报告页面 | 营养指标展示 + 柱状图 |
| 营养报告导出 | PDF下载 |
| fl_chart 图表集成 | 热量/蛋白质等日趋势图 |

### 交付物
- 基于菜单一键生成采购清单，按分类展示为Todo清单，支持勾选已购
- 基于菜单产出营养评估报告并导出PDF

---

## 8.5 Phase 4: 打磨上线

**目标：** UI/UX优化，性能调优，多平台测试

### 任务
| 任务 | 说明 |
|------|------|
| UI/UX 优化 | 统一视觉风格、动效、交互动画 |
| 空状态/错误处理 | 各页面空状态提示 + API错误处理 |
| 数据校验 | 输入校验提示 + 菜品配置完整性检查 |
| 性能优化 | 列表懒加载、食材下拉搜索优化 |
| Web 适配 | 响应式布局 + PWA支持 |
| Android 测试 | 真机测试 + 打包 |
| iOS 测试 | 模拟器/真机测试 |
| Web 部署 | 静态部署验证 |

### 交付物
- iOS / Android / Web 三端可用
- App包可发布

## 8.6 后端项目结构

```
backend/
├── app/
│   ├── main.py                 # FastAPI 应用入口
│   ├── config.py               # 配置加载
│   │
│   ├── api/
│   │   ├── __init__.py
│   │   ├── router.py           # 统一路由注册
│   │   ├── dishes.py           # 菜品API
│   │   ├── nutrition.py        # 营养分析API
│   │   ├── shopping.py         # 采购清单API
│   │   └── health.py           # 健康检查
│   │
│   ├── models/
│   │   ├── __init__.py
│   │   ├── database.py         # MySQL连接 + Session
│   │   └── dish_template.py    # dish_templates ORM模型
│   │
│   ├── schemas/
│   │   ├── __init__.py
│   │   ├── dish.py             # 菜品请求/响应Schema
│   │   ├── nutrition.py        # 营养分析Schema
│   │   └── shopping.py         # 采购清单Schema
│   │
│   ├── services/
│   │   ├── __init__.py
│   │   ├── llm_service.py      # LLM统一调用服务
│   │   ├── dish_service.py     # 菜品业务逻辑
│   │   ├── nutrition_service.py# 营养分析业务逻辑
│   │   └── shopping_service.py # 采购清单业务逻辑
│   │
│   └── utils/
│       ├── __init__.py
│       ├── prompts.py          # LLM Prompt模板
│       └── pdf_export.py       # PDF生成工具（营养报告导出用）
│
├── config.yaml                 # 配置文件
├── requirements.txt
└── README.md
```

### config.yaml 示例
```yaml
server:
  host: "0.0.0.0"
  port: 8000

mysql:
  host: "127.0.0.1"
  port: 3306
  user: "menu"
  password: "menu123"
  database: "menu"
  pool_size: 10

llm:
  provider: "openai"          # openai / ollama / custom
  api_base: "https://api.openai.com/v1"
  api_key: "sk-xxxxxxxx"
  model: "gpt-4o"
  # 或 Ollama:
  # provider: "ollama"
  # api_base: "http://localhost:11434/v1"
  # api_key: "ollama"
  # model: "qwen2.5:7b"
```
