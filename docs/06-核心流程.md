```
┌─────────┐          ┌──────────┐          ┌─────────┐

     │ 1.输入菜品名称      │                     │
     │───────────────────►│                     │
     │                    │                     │
     │                    │ 2.POST /api/dishes/generate
     │                    │────────────────────►│
     │                    │                     │
     │                    │                     │ 3.构造Prompt
     │                    │                     │ 4.调用LLM
     │                    │                     │ 5.解析结果
     │                    │                     │ 6.存入MySQL
     │                    │                     │
     │                    │ 7.返回菜品配置        │
     │                    │◄────────────────────│
     │                    │                     │
     │ 8.展示配置供确认     │                     │
     │◄───────────────────│                     │
     │                    │                     │

     │───────────────────►│                     │
     │                    │                     │
     │                    │ 10.保存到SQLite      │
     │                    │ (foods表)           │
     │                    │                     │
     │ 11.创建成功          │                     │
     │◄───────────────────│                     │
```

### LLM Prompt 设计思路

**生成食材配置：**
```
你是一个专业的厨师和营养师。请为以下菜品生成标准食材清单和营养数据。

菜品名称：{name}
分类：{category}
份量标准：该菜品的一份

请返回JSON格式：
{
  "ingredients": [{"name": "食材名", "quantity": 数量, "unit": "单位(g/ml/个/勺等)"}],
  "nutrition": {"calories": 热量kcal, "protein": 蛋白质g, "fat": 脂肪g, "carbs": 碳水g, "fiber": 膳食纤维g, "sodium": 钠mg}
}

注意：
1. 食材用量为一份菜品的标准用量
2. 营养数据为每份的估计值
3. 食材名称使用常见称呼，如"鸡胸肉"而非"鸡脯肉"
```

## 6.2 每周菜单创建流程

```
用户打开App
  │
  ▼
首页：本周菜单概览
  │
  ├── 有菜单 → 点击"查看/编辑" → 进入菜单编辑页
  │
  └── 无菜单 → 点击"新建菜单" → 选择周次 → 进入菜单编辑页
                                          │
                                          ▼
                              ┌─────────────────────┐
                              │   菜单编辑页         │
                              │                     │
                              │  周一 ┌──────────┐  │
                              │  主食:│下拉选择   ▼│  │
                              │       │ - 米饭    │  │
                              │       │ - 馒头    │  │
                              │       │ + 新增... │  │
                              │       └──────────┘  │
                              │  菜品:┌──────────┐  │
                              │       │下拉选择   ▼│  │
                              │       │ - 宫保鸡丁 │  │
                              │       │ + 新增... │  │
                              │       └──────────┘  │
                              │  + 添加菜品/主食    │
                              │                     │
                              │  周二 ... 同上      │
                              │  ...                │
                              │  周日 ... 同上      │
                              │                     │
                              │  就餐人数: [___]    │
                              │                     │
                              │  [保存] [取消]      │
                              └─────────────────────┘
```

### 下拉数据源逻辑
1. 读取本地 `foods` 表，按 `food_type` 过滤
2. 主食输入框 → 读取 `food_type = 'staple'` 的食物
3. 菜品输入框 → 读取 `food_type = 'dish'` 的食物
4. 若输入新名称（不在下拉列表中）→ 自动创建食物记录（仅有name和food_type，后续可补充食材配置）

## 6.3 采购清单生成流程

```
用户选择菜单 + 确认人数
  │
  ▼
Flutter 收集菜单中所有菜品的食材配置
  │
  │ (若本地菜品缺少食材配置 → 提示"部分菜品缺少食材配置，建议先完善"）
  │
  ▼
POST /api/shopping/generate
  发送：菜单菜品数据 + 人数
  │
  ▼
后端计算：
  1. 遍历所有菜品食材
  2. 按食材名称合并，用量 × 人数系数
  3. 自动归类到食材分类（肉类/蔬菜/干货/调料等）
  4. 按分类分组排序
  │
  ▼
返回分类分组的采购清单数据
  │
  ▼
Flutter 按分类展示为Todo清单：
  ┌─ 肉类 ───────────────────┐
  │ ☐ 鸡胸肉   600g          │
  │ ☐ 猪肉     400g          │
  ├─ 蔬菜 ───────────────────┤
  │ ☐ 黄瓜     200g          │
  │ ☐ 青椒     300g          │
  ├─ 干货 ───────────────────┤
  │ ☐ 花生米   100g          │
  └──────────────────────────┘
  │
  ▼
用户勾选已购买的食材项（勾选状态存前端内存，不持久化）
```

### 人数系数说明
- 人数系数：菜品食材用量 × (就餐人数 / 标准份数)
- 默认标准份数为1份，若菜单中某菜品设为2份，则再乘2

## 6.4 营养评估流程

```
用户选择菜单 → 点击"营养分析"
  │
  ▼
Flutter 收集菜单中所有菜品的营养数据
  │
  │ (若菜品缺少营养数据 → 提示/跳过)
  │
  ▼
POST /api/nutrition/analyze
  发送：菜单营养数据汇总
  │
  ▼
后端计算：
  1. 按天汇总营养摄入
  2. 计算日均值
  │
  ▼
POST /api/nutrition/assess (自动调用)
  发送：营养统计结果
  │
  ▼
后端调用LLM评估：
  - 输入：营养数据 + 推荐摄入标准
  - 输出：评估总结、亮点、风险、建议
  │
  ▼
返回完整报告数据
  │
  ▼
Flutter 展示：图表 + 文字评估
  │
  ▼
[导出PDF] 生成营养报告PDF
```
